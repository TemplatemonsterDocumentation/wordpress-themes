Введение

Благодарим Вас за приобретение шаблона WordPress. Данная документация состоит из нескольких частей и охватывает весь процесс установки и настройки веб-сайта WordPress с нуля.

Что представляет собой WordPress CMS?

WordPress - это бесплатное программное обеспечение с открытым исходным кодом для создания блогов и система управления контентом (CMS) на основе PHP и MySQL. Она позволяет создавать веб-сайты и мощные онлайн приложения, а также почти не требует технических навыков или знаний в управлении. Множество особенностей, в том числе простота использования и расширяемость, сделали WordPress наиболее популярной программой для веб-сайтов. Детали

Что такое шаблон WordPress

Шаблон WordPress - это тема для платформы WordPress CMS. Другими словами, вы можете легко изменить внешний вид веб-сайта WordPress путем установки нового шаблона за несколько несложных шагов. При всей своей простоте, шаблон WordPress содержит все необходимые исходные файлы, которые можно редактировать желаемым образом.

Структура шаблона

Приобретенный комплект шаблона состоит из нескольких папок. Проверим содержимое каждой папки:

  • documentation - содержит файлы документации.
    • documentation.html - основной файл документации. Вы его сейчас читаете :)
  • screenshots - содержит скриншоты шаблона. Для работы с шаблоном они не требуются.
  • sources.zip - содержит исходные файлы темы.
    • psd - исходные файлы шаблона для Adobe Photoshop(.psd).
  • theme - содержит файлы темы WordPress.
    • CherryFramework.zip - архив с темой Cherry Framework. Включает все файлы темы. Необходимо устанавливать через менеджер расширений WordPress.
    • theme####.zip - архив с темой (дочерняя тема). Содержит все файлы темы. Необходимо устанавливать через менеджер расширений WordPress.
    • sample_data - содержит файлы, которые используются для того чтобы сделать сайт WordPress таким же, как наше демо.
      • sample_data.xml - включает все демо-данные шаблона (записи, страницы, категории и т.п.).
      • widgets.json - файл настроек виджетов.
      • различные изображения и другие медиа файлы
    • manuall_install - содержит файлы, которые используются для того чтобы сделать сайт WordPress таким же, как наше демо.
      • uploads - содержит изображения темы.
      • theme#####.sql - файл базы данных (вмещает контент темы).
  • license.txt - файл с информацией о лицензии.
  • fonts_info.txt - файл со ссылками на скачку шрифтов, используемых в шаблоне.
  • info.txt - инструкции по разархивированию исходных файлов.

Установка шаблона

Установка шаблона еще никогда не была настолько проста. Процесс установки включает в себя всего три шага: установка Cherry Framework, установка дочерней темы и установка демо-данных.

Все этапы установки выполняются в админ-панели WordPress. Это делает процесс удобным и быстрым.

Вы можете убедиться в этом нажав на ссылку ниже:

Шаг 1. Установка CherryFramework

  1. Войдите в панель управления WordPress (добавьте /wp-admin к доменному имени сайта в адресной строке браузера).
  2. Перейдите в меню Appearance (Внешний вид) > Themes (Темы) и щелкните кнопку Add New Theme (Добавить новую тему).
  3. Откройте вкладку Upload (Загрузка).
  4. Нажмите кнопку Browse (Обзор) и откройте разархивированную папку шаблона на компьютере.
  5. Найдите файл CherryFramework.zip. Кликните на него и нажмите OK.
  6. Щелкните кнопку Install Now (Установить) и дождитесь окончания установки.
  7. Нажмите ссылку Activate (Активировать).

Для более детальной информации воспользуйтесь видео туториалом Как установить шаблон Cherry framework.

Поздравляем, вы успешно установили фреймворк.

Шаг 2. Установка темы

Теперь можно перейти к установке дочерней темы: theme####.zip. Повторите аналогичные шаги:

  1. Перейдите в меню Appearance (Внешний вид) > Themes (Темы) и щелкните кнопку Add New Theme (Добавить новую тему).
  2. Откройте вкладку Upload (Загрузка).
  3. Нажмите кнопку Browse (Обзор) и откройте разархивированную папку шаблона на компьютере.
  4. Найдите файл theme####.zip (где XXXX - уникальный номер темы). Кликните на него и нажмите OK.
  5. Щелкните кнопку Install Now (Установить) и дождитесь окончания установки.
  6. Нажмите ссылку Activate (Активировать), чтобы поменять текущий вид сайта на приобретенную тему.

Поздравляем, вы успешно установили фреймворк и тему!

Шаг 3. Установка демо-данных

Чтобы сайт WordPress выглядел как на нашем демо, воспользуйтесь следующими инструкциями. Мы рекомендуем всегда делать резервное копирование файла .sql через экспорт базы данных.

ВНИМАНИЕ: Для шаблонов на базе Cherry Framework v3.0 или меньше при установке демо-данных требуется загрузить файлы XML и JSON из папки "sample_data". Изображения и другие файлы мультимедиа нужно загрузить из папки "manual_install\uploads". Загрузите все картинки из всех подпапок.
  1. Перейдите в меню Cherry Plugin > Import Content (Импорт контента).
  2. Прежде всего, проверьте таблицу требований Cherry Plugin к настройкам хостинг сервера. В случае появления предупреждений, обратитесь к хостинг провайдеру.
  3. Если все настройки сервера в порядке, появится экран загрузки. Откройте папку "theme/sample_data" шаблона и загрузите все файлы в WordPress. Все файлы в папке "theme/sample_data" нужно выделить (горячие клавиши CTRL+A или Command+A) и перетянуть в поле загрузки.
  4. Загрузите все файлы из папки sample_data, в том числе .JSON и .XML. По завершению загрузки нажмите кнопку "Continue Install (Продолжить установку)".

    ВНИМАНИЕ: Файлы .JSON и .XML являются обязательными при установке демо-данных. Кнопка Continue Install (Продолжить установку) станет активна только после загрузки файлов .XML и .JSON.
  5. Кнопка "Continue Install (Продолжить установку)" начнет процесс установки демо-данных. Установка займет определенное время в зависимости от настроек сервера. Пожалуйста, дождитесь ее окончания.
  6. Поздравляем! Вы успешно установили тему Cherry WordPress и теперь можете проверить, как она выглядит.

Подготовка

Прежде чем перейти к установке веб-сайта WordPress, нужно полностью подготовиться. Рекомендуется выполнить следующие подготовительные шаги:

Программное обеспечение

Для полноценной работы с шаблоном WordPress мы рекомендуем скачать все требуемые приложения. Список необходимого программного обеспечения находится на странице предпросмотра шаблона.
Требования для различных шаблонов могут меняться, поэтому мы перечислим самые основные:

  1. Прежде всего нужны правильные приложения для распаковывания защищенного паролем архива sources_#########.zip. Можно воспользоваться WinZip 9+ (Windows) и Stuffit Expander 10+ (Mac).
  2. Также может понадобиться программа Adobe Photoshop. Она используется для редактирования исходных файлов .PSD и необходима, если требуется изменить графический дизайн и изображения шаблона.
  3. Для редактирования исходного кода файлов шаблона нужно использовать редактор кода, например Adobe Dreamweaver, Notepad++, Sublime Text и т.п.
  4. Чтобы загрузить файлы на хостинг сервер, потребуется FTP менеджер, например Total Commander, FileZilla, CuteFTP и т.д.

Хостинг

Поскольку WordPress CMS является приложением PHP/MySQL, необходимо подготовить хостинг сервер.

При наличии активного сервера, нужно проверить его на соответствие требованиям WordPress и на готовность использовать веб-сайты WordPress.

Также можно установить WordPress на компьютер при помощи локального сервера. Для создания локального хостинг сервера необходимо использовать программное обеспечение WAMP, AppServ, MAMP и т.п. Любое из них устанавливается как обычная программа и поддерживает WordPress.

Ручная установка

В случае возникновения сложностей при установке демо-данных воспользуйтесь следующими инструкциями.

ВНИМАНИЕ: Импорт файла SQL в базу данных перезапишет существующий контент и настройки веб-сайта. НЕ импортируйте файл SQL, если нужно сохранить текущий контент.
ПАМЯТКА: Всегда делайте резервное копирование базы данных, прежде чем приступить к изменениям.
Префикс таблиц базы данных WordPress должен быть wp_.

Чтобы веб-сайт WordPress выглядел как на демо, следуйте данным инструкциям.

  1. Тема должна быть установлена согласно следующим указаниям.
  2. Скопируйте папку "uploads" (она находится в папке theme/manual_install/ шаблона) в папку /wp-content/ на FTP сервере.
  3. Откройте файл theme####.sql из папки theme/manual_install/ в любом текстовом редакторе (рекомендуется Dreamweaver или Notepad) и замените все строки "your_website_url_here" на URL вашего веб-сайта в документе при помощи опции Find and Replace (Найти и Заменить) (окно вызывается горячими клавишами Ctrl+H). Например: http://www.mywebsite.com
  4. В конце адреса не должно быть знака косой черты "/" и url должен начинаться с http://www.
  5. Сохраните изменения и закройте файл.
  6. Теперь файл дампа можно импортировать через phpMyAdmin или другой инструмент управления базами данных.

В случае возникновения сложностей при установке через панель управления, можно воспользоваться альтернативным способом.
Загрузка через FTP:

Для использования данного метода, нужно иметь доступ к файлам WordPress на FTP сервере через файловый менеджер контрольной панели хостинга или FTP-клиент, например Filezilla, CuteFTP, Total Commander, и т.п.

Если вы установили тему с демо-данными или вручную, а тема отличается от демо, откройте Cherry Options (Опции Cherry) и нажмите "Restore Defaults (Восстановить настройки по умолчанию)."
  1. Разархивируйте файлы CherryFramework.zip и theme####.zip в папку на компьютере (щелкните правой кнопкой по каждому файлу.zip, выберите Unzip to… (Распаковать в ...) CherryFramework и theme#### соответственно, после чего появятся две папки: CherryFramework и theme####).
  2. Загрузите папки CherryFramework и theme### в директорию /wp-content/themes/ на FTP сервере.
  3. Войдите в панель управления WordPress (добавьте /wp-admin к доменному имени сайта в адресной строке браузера).
  4. Перейдите в меню Appearance (Внешний вид) > Themes (Темы).
  5. В разделе Available Themes (Доступные темы) найдите theme#### и активируйте ее, щелкнув по кнопке Activate (Активировать).

Установка плагинов

После установки и активации темы иногда требуется установить дополнительные плагины:

  1. В панели управления WordPress перейдите в меню Plugins (Плагины) -> Add New (Добавить новый).
  2. Нажмите Upload (Загрузить).
  3. Щелкните кнопку Browse (Обзор).
  4. Откройте папку theme####.zip\includes\plugins шаблона (файл theme####.zip должен быть разархивирован).
  5. Выберите один из файлов .zip в этой папке и нажмите OK.
  6. Кликните на кнопку Install Now (Установить).
  7. По окончанию установки плагина нажмите Activate (Активировать).
  8. Если в папке /plugins/ находятся несколько плагинов, повторите шаги по установке и активации каждого из них.

Плагин Cherry

Скачать: https://github.com/CherryFramework/CherryFramework/raw/master/includes/plugins/cherry-plugin.zip

Он представляет собой расширение для нашего фреймворка Cherry, в котором мы добавили все необходимые шорткоды и виджеты. Плагин полностью совместим с любой темой WordPress на основе Cherry Framework. Политика кукис

Куки представляют собой небольшие текстовые файлы, которые веб-сайты создают на компьютерах пользователей. Они в основном используются для идентификации конкретного посетителя и обеспечения более высокой производительности. Куки являются важной частью многих функций веб-сайта.

Например, в онлайн магазинах куки запоминают, что именно вы добавили в корзину. Они также сохраняют авторизацию на сайте и обеспечивают владельцев веб-сайтов ценной статистикой и информацией.

Если у вас есть веб-сайт, скорее всего он использует куки. Наиболее основными функциями являются наблюдение за количеством посетителей и использованием инструментов, например Google Analytics. Они также могут показывать важные сообщения пользователям или, если у вы занимаетесь онлайн продажами, быть мощной частью электронного магазина.

26-го мая 2012 - ICO внесло поправку в закон о кукис и теперь скрытое содержание разрешается только при уведомлении о действии приватности/политики куков на сайте.

Согласно Закону о Кукис необходимо добавлять на веб-сайт подходящий модуль, показывающий сообщение об использовании кукис. Можно воспользоваться следующими плагинами: Cookie Banner, Cookie Notice

Резервное копирование и обновление

Перед обновлением Cherry Framework, мы убедительно советуем сделать резервную копию текущей версии.

Резервное копирование

Можно сделать резервную копию как CherryFramework, так и дочерней темы в меню "Cherry Options (Опции Cherry) > Data management (Управление данными)" панели управления WordPress.

Резервную копию легко создать, нажав кнопку "Backup" под секциями с Cherry Framework и дочерней темой.
После этого архив с фреймворком или дочерней темой появится в папке /wp-content/themes_backup/.

Если нужно его скачать, кликните кнопку "Download backup". В случае возникновения сложностей, фреймворк можно переустановить во вкладке Appearance (Внешний вид)-> Themes (Темы)-> Install Themes (Установить темы)-> Upload (Загрузка).

Чтобы обновить Cherry Framework, сделайте следующее:

  1. В панели управления откройте вкладку "Cherry Options (Опции Cherry)> Data management (Управление данными)".
  2. Нажмите кнопку Update (Обновить) под блоком Cherry Framework
  3. Дождитесь начала обновления.
  4. Все остальное Cherry Framework выполнит автоматически.

Стандартный формат записи

Обычная, стандартная запись имеет следующие поля:

  • Post title - заголовок записи
  • Post content - контент записи
  • Categories - категория (и) записи
  • Tags - теги записи
  • Featured Image - изображение записи
  • и другие поля, которые можно включить в разделе Screen Options (Настройки экрана) в верхнем правом углу.

Стандартный формат записи

Заметка

Представляет собой упрощенную стандартную версию записи. Другими словами, в ней находится только поле для контента, поэтому не требуется вводить заголовок записи и другие опции.

Заметка

Галерея

Основной особенностью данного формата записи является возможность добавления неограниченного количества изображений.

Добавление галереи:

  1. Нажмите кнопку Add Media (Добавить медиа).
  2. Чтобы добавить изображения с компьютера, перетяните файлы в появившееся поле или щелкните Select Files (Обзор).
  3. После загрузки все картинки находятся в Media Library (Медиа библиотека). Выберите Uploaded to this post (Добавленные в эту запись) для просмотра изображений.
  4. По окончанию редактирования можно закрыть медиа библиотеку. Загруженные картинки будут добавлены автоматически.

В записях формата галерея находятся те же поля (например, заголовок, контент, категории, и т.п.), что и для стандартного формата записи.

Ссылка

Формат Ссылка содержит дополнительное поле URL, где нужно указать ссылку (URL) для целевой страницы/ресурса. Заголовок записи будет использоваться в качестве текста ссылки.

Поле контента представляет собой описание целевой страницы/ресурса.

Изображение

Изображение - это формат записи в виде одиночного изображения, загруженного через опцию Featured Image (Изображение записи).
Выбрав данный формат, вы сможете отключить лайтбокс для изображений при помощи соответствующей настройки Enable Lightbox (Включить лайтбокс) .

Цитата

Пост Цитата используется для публикации высказываний.

Данный формат отличается наличием дополнительный полей Цитата - текста высказывания и Автор - имя автора цитаты.

Аудио

Формат Аудио используется для публикации аудио-контента.

Отличается наличием следующих настроек:

  • Artist (Исполнитель) - исполнитель аудиотрека

Видео

Видео используется для публикации видео-контента.

Отличается наличием следующих настроек:

  • Embedded Code (Встроенный код) - используется для добавления видео с внешних ресурсов, например, youtube, vimeo, и т.д.

Шорткоды - мощный инструмент управления контентом http://codex.wordpress.org/Shortcode. Они очень просты в использовании. Прежде всего, нужно выбрать режим редактирования Visual (Визуальный).

Все шорткоды условно делятся на следующие группы:

Audio / Video (Аудио / Видео)

Описание: WordPress 3.6+ отлично поддерживает добавление видео с YouTube и Vimeo, но если нужно воспроизвести видео или аудиофайл в формате MP3 или MP4, загруженный на сервер, потребуется установить плеер, который будет играть роль проигрывателя, а не ссылки на сайте. Подробнее

Процесс аналогичен добавлению изображения в контент записи:
  1. Добавьте или отредактируйте страницу или запись.
  2. Щелкните по контенту, где нужно добавить плеер, затем нажмите Add Media (Добавить медиа) над редактором контента.
  3. Щелкните Upload Files (Загрузить файлы).
  4. Загрузите аудио или видео файл.
  5. Выделите загруженный файл, затем в Attachment Display Settings (Настройки отображения дополнений), в опции Embed or Link (Вставить или ссылка) выберите "Embed Media Player (Встроенный медиа плеер)."
  6. Нажмите Insert into page (Добавить на страницу).

WordPress добавил шорткод для аудио или видео, которые заменятся на проигрыватель.

Если вы добавили аудио или видеофайл на другой сервер, например Amazon S3 или файловый хостинг, вставьте URL в медиа файл (без ссылки) в соответствующее поле. Например, нужно загрузить видео с YouTube. Просто скопируйте его в соответствующее поле и проигрыватель YouTube player появится там автоматически. Убедитесь, что URL задан правильно.

Shop Beginners Kit

Важно! Shop Beginners Kit работает только на Cherry Framework версии 2.0 и выше.
Если ваша версия ниже, чем 2.0, обновите Cherry Framework.

Shop Beginners Kit можно приобрести вместе с темой WordPress или отдельно.

Прежде всего необходимо установить и активировать плагин WooCommerce или Jigoshop.

Плагины можно скачать по следующим ссылкам:

Плагин Jigoshop: http://wordpress.org/plugins/jigoshop/
Плагин WooCommerce: http://wordpress.org/plugins/woocommerce/

Внимание! Для полноценного функционирования требуется лишь один плагин.
Установка плагина
  1. В панели управления WordPress, перейдите в меню Plugins (Плагины) -> Add New (Добавить новый).
  2. Щелкните кнопку Upload (Загрузить).
  3. Нажмите Browse (Обзор).
  4. Выберите файл одного из плагинов и нажмите OK.
  5. Кликните Install Now (Установить).
  6. После установки плагина, нажмите Activate (Активировать).
Если вы приобрели данное предложение вместе с темой WordPress, нужно выполнить несколько шагов для его корректной работы.

Также вы можете установить виджеты и демо товары для вашего магазина.

Необходимо сделать следующее:

  1. Установка виджетов:
    • В меню выберите Cherry Options (Опции Cherry) -> Import (Импорт)
    • Выберите файл jigoshop_widgets.json или woocommerce_widgets.json в зависимости от того, какой плагин вы установили. После этого нажмите кнопку Show Widget Settings (Показать настройки виджета) (данный этап можно пропустить и настроить все виджеты вручную в меню Appearance (Внешний вид) -> Widgets (Виджеты)).
    • Выберите виджеты для установки и затем щелкните кнопку Import Settings (Импорт настроек).
    • Дождитесь окончания установки.
  2. Добавление демо товаров.
    • Выберите файл shop_sample_data.xml и нажмите кнопки Upload file (Загрузить) и Import (Импорт) (данный этап можно пропустить и выполнить позднее).
    • В поле Import Author (Автор импорта), выберите пользователя и нажмите кнопку Import Data (Импорт данных).
    • После установки демо товаров, нажмите кнопку Finish (Завершить). Теперь ваш магазин готов к дальнейшей работе.

Если вы решили добавить функционал магазина к раннее приобретенному шаблону, необходимо получить дополнительные файлы, которые обеспечивают совместимость Cherry Framework с плагинами WooCommerce и Jigoshop.

  2. Из папки theme#### скопируйте два файла (shop.less и shop.php) в корневую папку с дочерней темой.
